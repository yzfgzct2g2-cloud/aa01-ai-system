## 二、尚未完成（與標準模板的落差）

1. **一、(五) 輔具及居家環境概況**：標準模板要求獨立成段；目前環境內容併入「一、(一)4. 居家環境與社會參與」，未獨立。
2. **一、(三) 經濟概況**：無資料欄位，永遠佔位。
3. **一、(四) 社會支持概況**：H 的社會參與資料未對應到此段（目前環境/社會混在一起）。
4. **二、(五) 營養餐飲服務**：標準模板新增分類，產生器尚無此段。
5. **四、營養餐飲分類**：核定服務內容缺營養餐飲類。
6. **七、退件紀錄**：標準模板為「送審與退件紀錄」，目前無退件區塊。
7. **段落非固定**：目前服務段落（四之一～四）採「有資料才輸出」；標準模板要求**段落固定存在**、缺資料顯示「待補充／目前無相關需求」。
8. **系統輔助提示**穿插於三與四之間，屬輔助資訊，正式版應移出正式段落或明確標示。

## 三、需要 Service Goal Library 的部分

- **三、計畫執行規劃（短/中/長期目標）**：目前為規則式樣板句，未依個案實際服務碼客製。需建立「服務碼 → 核定內容 / 計畫目標」對照（如 BA13、BA14、BA15-2、DA01…），由 Library 產生對應目標語句。
- **四、擬核定照顧服務內容的核定內容描述**：每個服務碼的正式核定文字亦應來自 Library。

## 四、需要 Reviewer Preference 的部分

- **七、送審與退件紀錄**：退件原因、修正後版本、各照專偏好版型。目前無退件案例，僅有 [退件紀錄格式](../05-reviewer-preferences/退件紀錄格式.md)。

## 五、其他待補欄位（資料面，非產生器邏輯）

- 經濟概況、社會支持、轉介資源、落差（照專建議／案家意願）、送審（個管／督導／日期）目前皆無結構化欄位，需於 `types.ts` / 表單補充後才能脫離佔位。
- 家庭功能概況需結構化「家庭成員」而非單一備註。

## 六、建議重構順序（不在本次範圍）

1. 以標準模板的**固定段落骨架**重寫 `buildAA01Draft`（缺資料顯示「待補充／目前無相關需求」）。
2. 拆分「居家環境」與「社會支持」，補上一、(五) 與二、(五)。
3. 導入 Service Goal Library 產生三與四的正式語句。
4. 最後接 Reviewer Preference 補七、退件紀錄。
