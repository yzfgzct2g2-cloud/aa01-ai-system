# AA01 Knowledge Base

本資料夾為 AA01 照顧計畫輔助系統的知識庫，供系統規則、文風學習與範例累積使用。
所有內容皆為「個管確認後的真實資料」或「系統開發規範」，不得放入虛構或臆測內容。

## 資料夾結構

```
knowledge/
├─ 01-official/            正式法規、照顧管理評估量表、支付基準
├─ 02-service-rules/       BA/BB/BC/BD/DA/EA/FA/SC 等服務規則與限制
├─ 03-plan-examples/       已通過之 AA01 範例（含退件後修正版）
├─ 04-care-manager-style/  個管個人撰寫風格、常用語句、不喜歡的語句
├─ 05-reviewer-preferences/ 照專偏好、退件原因與修正習慣
├─ 06-templates/           新案、複評、半年獨訪、異動等模板
└─ 99-project-rules/       系統開發規則、Roadmap、AI 限制原則
```

## 各資料夾用途

- **01-official**：正式法規、評估量表、支付基準（唯讀來源，不可改寫原始選項）。
- **02-service-rules**：各服務碼對應規則與限制。
- **03-plan-examples**：AA01 通過案例、退件案例與修正版（目前已有代表性案例）。
- **04-care-manager-style**：個管個人撰寫風格與常用語句。
- **05-reviewer-preferences**：照專偏好、退件原因與修正方向（目前尚無退件案例，僅先建立格式）。
- **06-templates**：各類 AA01 輸出模板。
- **99-project-rules**：系統開發規則與 AI 工作規範，見 `ROADMAP.md` 與 `DEVELOPMENT_RULES.md`。

## 資料收集策略

AA01 範例不是越多越好，而是要有「代表性」。目前範例已可使用，後續逐步累積下列類型：

1. 純居服案
2. 居服 + 交通接送
3. 居服 + 專業服務
4. 居服 + 輔具
5. 喘息服務
6. 失智症個案
7. 獨居個案
8. 重度失能個案
9. 吞嚥問題個案
10. 壓傷或皮膚照護個案
11. 家庭支持薄弱個案
12. 外籍看護個案

> 目前無退件案例，請勿建立虛構退件規則，僅先維護退件紀錄格式。
